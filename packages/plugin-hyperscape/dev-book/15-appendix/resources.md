## Getting Help

### Support Channels
1. **Documentation**: Read this dev-book first
2. **FAQ**: Check [FAQ](faq.md) for common questions
3. **GitHub Issues**: Report bugs or request features
4. **Discord**: Join for community support
5. **Email**: support@hyperscape.ai (Coming soon)

### Reporting Bugs
When reporting bugs, include:
- Plugin version
- ElizaOS version
- Error messages and stack traces
- Steps to reproduce
- Environment (OS, Node version, etc.)

### Feature Requests
When requesting features:
- Describe the use case
- Explain why it's valuable
- Suggest implementation approach (optional)
- Provide examples (optional)

## Contributing

### How to Contribute
1. Fork the repository
2. Create a feature branch
3. Write tests (real tests, no mocks!)
4. Submit a pull request

### Contribution Guidelines
- Follow TypeScript strict typing (no `any`)
- Write comprehensive tests
- Update documentation
- Follow code style (ESLint)

See [Development Guide](../11-development/setup-guide.md) for details.
